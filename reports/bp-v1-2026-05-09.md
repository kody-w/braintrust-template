## TL;DR

**Bond Pulse** is the on-going local↔global heartbeat for the FULL RAPP organism. ONE organism, TWO body parts (global = offspring repos; local = `~/.brainstem/`), ONE heartbeat. Each pulse: audit drift → classify direction → SUGGEST actuator (operator-mediated, never auto-executes) → record `kind="rhythm"` bond event → return `rapp-rhythm-pulse/1.0`. Connection-aware: degrades to local-only when offline; the next pulse with connection catches the body up.

## What it preserves

1. **Both halves of the organism stay reconciled** without either side clobbering the other (bond technique: egg → overlay → hatch back).
2. **Operator agency** — the heartbeat SUGGESTS but never ACTUATES. Operator drives Launch / Graft / RarLoader explicitly. `dry_run=True` is enforced on the rhythm agent regardless of what the caller passes.
3. **Audit trail integrity** — every pulse records a `kind="rhythm"` event in `~/.brainstem/bonds.json` (append-only). Suggested actions and dispatched actions are separable in the log.
4. **Connection-aware progress** — the colony of organisms doesn't stop pulsing when offline; each `degraded=True` pulse captures local state and the next online pulse catches up.

## Why operator-mediated (never auto-execute)

Three load-bearing reasons (vault note, §"Why operator-mediated"):

1. **"Who's right" when global+local diverge is a judgment call only the operator can make.** The audit knows what's different; only the operator knows what's RIGHT.
2. **Connection-aware degradation requires human latency anyway.** If a different process is auto-firing actuators in the background, the catch-up queue from offline pulses becomes meaningless.
3. **Actuators all use the bond technique already** (preserve-local + append-only bond events). Auto-firing isn't UNSAFE — it just dilutes the audit trail with rhythm-driven events the operator never chose. Operator-mediated keeps `bonds.json` honest.

## Connection-aware degradation

```
def pulse_once(allow_online=False):
    audit = run_audit(online=allow_online)        # may fail / timeout
    if audit_failed:
        degraded = True
        audit = empty_envelope                     # 0 offspring, 0 drift
    classify(audit) → suggested_actions
    record bonds.json event(kind="rhythm", degraded, drift_count, ...)
    return rapp-rhythm-pulse/1.0 envelope
```

The pulse always returns a valid envelope. `degraded=True` flags "I couldn't see the global body this time." The next pulse with connection sees full state and the body catches up — no data loss, no clobbering. Implementation: `bond_rhythm_agent.py::_run_audit_subprocess` catches `OSError`/timeout and returns `(None, error_msg)`; `BondRhythm._perform` sets `degraded=True` and continues with empty audit envelope. Test coverage: `tests/features/F12-bond-rhythm.sh::Step_9` ("bad repo_root → degraded=True + valid envelope").

> **Preserve a long-evolved local mutation; bring it back into the wider structure additively; record the act in an append-only log; never destroy what was already there.**
